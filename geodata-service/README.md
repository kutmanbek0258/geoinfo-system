# GeoData Service

## 1. Обзор

`geodata-service` — это ключевой микросервис в рамках "ГеоИнфоСистемы", отвечающий за хранение и управление пространственными данными и их метаданными. Он предоставляет централизованный API для работы с проектами, папками, векторными объектами, реестром растровых снимков (как проектных, так и глобальных) и слоями 3D-рельефа.

## 2. Основные обязанности

- **Управление проектами и доступом:** Создание проектов, управление списком пользователей с правом чтения/записи (`ProjectAccess`).
- **Иерархическая структура проектов:** Организация слоев и векторных объектов с помощью древовидных папок (`GeoFolder`).
- **Импорт KML:** Разбор KML-файлов с автоматическим созданием проектов и векторных слоев.
- **Управление геообъектами (Векторные данные):** Полный цикл CRUD-операций для векторных объектов:
    - **Точки** (`ProjectPoint`)
    - **Линии** (`ProjectMultiline`)
    - **Полигоны** (`ProjectPolygon`)
- **Универсальная система слоев:**
    - Базовая сущность **Слой** (`Layer`) с привязкой к проекту, описанием, типом и гибкими характеристиками (JSONB).
    - **Растровые снимки проекта** (`ProjectRaster`), привязанные к логическим слоям и папкам.
    - **Глобальные растровые слои** (`RasterLayer`) для общесистемных картографических подложек.
    - **3D-рельеф (Quantized Mesh DEM)** (`TerrainLayer`) — глобальные файлы рельефа для визуализации трехмерной поверхности.
- **Генерация Presigned URLs:** Прямое создание безопасных временных ссылок на скачивание/отображение файлов растров и рельефа из S3 (MinIO) минуя другие шлюзы.
- **Асинхронная индексация:** Отправка событий об изменении данных в Apache Kafka для мгновенной индексации в `search-service`.
- **Автоматические триггеры PostGIS:** Расчет длин линий и площадей полигонов на уровне СУБД.

## 3. Технологический стек

- **Фреймворк:** Spring Boot 3
- **Язык:** Java 17
- **База данных:** PostgreSQL с расширением PostGIS (схема `geodata`)
- **Управление миграциями БД:** Liquibase
- **Работа с геометриями:** Hibernate Spatial, JTS (Java Topology Suite)
- **Сборка:** Apache Maven

## 4. Модели данных

Сервис оперирует следующими основными JPA-сущностями:

- `Project`: Проект, объединяющий слои и объекты.
- `ProjectAccess`: Права доступа пользователей к проектам.
- `GeoFolder`: Узлы дерева каталогов внутри проекта.
- `ProjectPoint` / `ProjectMultiline` / `ProjectPolygon`: Векторные пространственные объекты с PostGIS геометриями.
- `Layer`: Сущность логического слоя в рамках проекта.
- `ProjectRaster`: Снимок (GeoTIFF COG) в проекте, привязанный к слою.
- `RasterLayer`: Глобальный общесистемный растр.
- `TerrainLayer`: Глобальный слой 3D-рельефа.
- `RasterStyle`: Стили оформления растровых данных (SLD).

Все гео-сущности привязаны к проекту и содержат колонку `geom` типа `Geometry` для хранения пространственных данных.

## 5. API Endpoints

Сервис предоставляет REST API по базовому пути `/api/geodata`.

### Проекты (`/project`)
- `POST /`: Создать проект.
- `POST /import/file`: Импортировать проект из KML или KMZ файла (параметры: `file`, `projectName` - необязательно).
- `GET /{id}`: Получить проект.
- `GET /page-query`: Получить список проектов.
- `PUT /{id}`: Обновить проект.
- `DELETE /{id}`: Удалить проект.
- `POST /{projectId}/share`: Поделиться проектом.

### Логические слои (`/layers`)
- `POST /`: Создать логический слой.
- `GET /{id}`: Получить слой по ID.
- `PUT /{id}`: Обновить слой.
- `DELETE /{id}`: Удалить слой.
- `GET /project/{projectId}`: Получить все слои проекта.

### Растровые снимки проекта (`/project-rasters`)
- `POST /`: Создать растровый снимок.
- `GET /{id}`: Получить снимок по ID.
- `PUT /{id}`: Обновить снимок.
- `DELETE /{id}`: Удалить снимок и его файлы в S3.
- `GET /layer/{layerId}`: Получить снимки слоя.
- `GET /folder/{folderId}`: Получить снимки папки.

### Глобальные растровые слои (`/raster-layers`)
- `POST /`: Создать глобальный растр.
- `GET /{id}`: Получить глобальный растр по ID.
- `PUT /{id}`: Обновить глобальный растр.
- `DELETE /{id}`: Удалить глобальный растр и его файлы в S3.
- `GET /`: Получить список всех глобальных растров.

### 3D Рельеф (`/terrain-layers`)
- `POST /`: Зарегистрировать обработанный слой 3D-рельефа.
- `GET /{id}`: Получить слой рельефа по ID.
- `GET /{id}/presigned-url`: Получить presigned-ссылку на COG рельефа.
- `PUT /{id}`: Обновить метаданные рельефа.
- `DELETE /{id}`: Удалить слой рельефа и его файлы в S3.
- `GET /`: Получить список всех слоев рельефа с пагинацией.

### Точки (`/points`)
- `POST /`: Создать точку.
- `GET /`: Получить все точки.
- `GET /{id}`: Получить точку по ID.
- `GET /by-project-id/{projectId}`: Получить все точки в проекте.
- `PUT /{id}`: Обновить точку.
- `DELETE /{id}`: Удалить точку.
- `POST /{id}/upload-main-image`: Загрузить изображение для точки.

### Линии (`/multilines`)
- `POST /`: Создать линию.
- `GET /`: Получить все линии.
- `GET /{id}`: Получить линию по ID.
- `GET /by-project-id/{projectId}`: Получить все линии в проекте.
- `PUT /{id}`: Обновить линию.
- `DELETE /{id}`: Удалить линию.
- `POST /{id}/upload-main-image`: Загрузить изображение для линии.

### Полигоны (`/polygons`)
- `POST /`: Создать полигон.
- `GET /`: Получить все полигоны.
- `GET /{id}`: Получить полигон по ID.
- `GET /by-project-id/{projectId}`: Получить все полигоны в проекте.
- `PUT /{id}`: Обновить полигон.
- `DELETE /{id}`: Удалить полигон.
- `POST /{id}/upload-main-image`: Загрузить изображение для полигона.

## 6. База данных

Схема базы данных управляется с помощью **Liquibase**.
- **Схема:** `geodata`
- **Расширения PostgreSQL:** `uuid-ossp`, `postgis`
- **Ключевые особенности:**
    - Пространственные **GIST-индексы** для всех колонок с геометрией для обеспечения высокой производительности гео-запросов.
    - **Триггеры** для автоматического расчета площади и длины.
    - Внешние ключи с каскадным удалением (`ON DELETE CASCADE`) от геообъектов к проектам.

Миграции находятся в директории `src/main/resources/db/changelog`.

## 7. Сборка и запуск

Для сборки и запуска сервиса локально используйте Maven:

```bash

mvn clean install

# Запуск сервиса
mvn spring-boot:run
```
