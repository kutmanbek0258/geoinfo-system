# Реестр активных плагинов GeoAnalysis Worker

В данном документе перечислены все функциональные модули (плагины), доступные в текущей сборке воркера, с описанием их входных данных и параметров.

---

## 1. Terrain Contours (Генерация изолиний)
**Идентификатор (`pluginName`):** `terrain_contours`

Плагин для извлечения векторных линий равных высот из цифровых моделей рельефа (DEM).

### Входные данные (`inputs`)
| Ключ | Описание | Формат |
| --- | --- | --- |
| `dem_file` | Исходный растр высот | S3 URL (GeoTIFF) |

### Параметры (`parameters`)
| Параметр | Описание | Тип | По умолчанию |
| --- | --- | --- | --- |
| `interval` | Шаг между изолиниями в метрах | float | `10.0` |
| `base` | Базовая высота для начала отсчета | float | `0.0` |
| `attribute` | Имя атрибута высоты в итоговом файле | string | `"elev"` |
| `use_3d` | Создавать ли 3D геометрию (Z-координату) | boolean | `false` |
| `format` | Формат выходного файла | string | `"GeoJSON"` |

### Результаты (`outputs`)
*   `vector_result`: S3 URL к созданному файлу изолиний.

---

## 2. Zonal Statistics (Зональная статистика)
**Идентификатор (`pluginName`):** `zonal_statistics`

Плагин для расчета статистических показателей растра в границах векторных зон.

### Входные данные (`inputs`)
| Ключ | Описание | Формат |
| --- | --- | --- |
| `source_raster` | Растр со значениями для анализа | S3 URL (GeoTIFF) |
| `zones_vector` | Векторный слой с полигонами зон | S3 URL (GeoJSON) |

### Параметры (`parameters`)
| Параметр | Описание | Тип | По умолчанию |
| --- | --- | --- | --- |
| `stats` | Список метрик (`min`, `max`, `mean`, `count`, `sum`, `std`, `median`) | array | `["mean", "count"]` |
| `categorical` | Считать ли количество каждого уникального значения | boolean | `false` |
| `add_properties` | Сохранять ли атрибуты исходного вектора | boolean | `true` |

### Результаты (`outputs`)
*   `statistics_json`: S3 URL к JSON-файлу с массивом статистических данных.
*   `statistics_geojson`: S3 URL к GeoJSON-файлу с геометрией и статистикой в атрибутах.

---

## Общие примечания
*   **Идемпотентность:** Все плагины перезаписывают результаты при повторном запуске с тем же `taskId`.
*   **Очистка:** Все промежуточные данные удаляются из RAM-диска сразу после загрузки результатов в S3.
*   **Проекции:** Плагины автоматически выполняют перепроецирование (Reproject) векторных данных к системе координат растра для обеспечения точности расчетов.
