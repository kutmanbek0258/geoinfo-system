# GeoPrint Worker (Python)

`geoprint-worker` — это сервис-воркер на Python в составе архитектуры **ГеоИнфоСистема**, выполняющий непосредственный рендеринг картографических слоев и компиляцию печатных PDF-отчетов. Воркер запускается асинхронно, получая задачи из очереди Apache Kafka, и выгружает готовые отчеты в S3-совместимое хранилище (MinIO).

## 🚀 Основные возможности

*   **Рендеринг растровых слоев через TiTiler:** Вместо прямого чтения тяжелых TIFF-файлов, воркер запрашивает фрагменты изображений с наложенными стилями по REST API TiTiler (`/cog/bbox/{minx},{miny},{maxx},{maxy}.png`).
*   **Отрисовка векторных слоев:** Скачивает GeoJSON-файлы векторных данных из S3, загружает их с помощью `geopandas` и накладывает поверх растра с учетом индивидуальных стилей (`strokeColor`, `strokeWidth`, `fillColor`, `fillOpacity`) и прозрачности.
*   **Высокоточное позиционирование (Matplotlib):** Масштабирование, обрезка под заданный экстент (BBOX) и репроекция геометрий с помощью картографической библиотеки `Cartopy`.
*   **Генерация PDF (ReportLab):** Сборка профессиональных многоформатных документов (от **A4 до A0**, альбомной/книжной ориентации) с добавлением рамок, штампов, метаданных (заголовок, автор, дата, организация) и интеграцией отрендеренной карты.

## 🛠 Технологический стек

*   **Python 3.9+**
*   **Matplotlib & Cartopy:** Отрисовка картографической сетки, растров и векторных геометрий.
*   **Geopandas & Shapely:** Чтение, обработка и проецирование географических данных.
*   **ReportLab:** Генерация высококачественных PDF-файлов.
*   **kafka-python:** Взаимодействие с брокером Apache Kafka.
*   **Minio Python SDK:** Загрузка и скачивание ресурсов из S3-хранилища.

## 📂 Процесс обработки задачи (Workflow)

1.  **Прием задачи:** Воркер слушает топик `geo.print.tasks`. При получении сообщения создается временная рабочая директория для промежуточных файлов.
2.  **Загрузка векторов:** Воркер сканирует слои из спецификации печати. Если обнаружены векторные слои (`VECTOR`) со ссылкой на S3 (`s3://...`), они скачиваются локально, парсятся и инлайнятся в спецификацию.
3.  **Рендеринг растра:** Для каждого растрового слоя (`COG`) воркер делает запрос к TiTiler, передавая параметры масштабирования (`colormap`, `colormap_name`, `resampling`) и BBOX. Полученный PNG-файл накладывается на холст Matplotlib.
4.  **Рендеринг векторов:** Векторные объекты отрисовываются поверх растра с использованием стилей слоя.
5.  **Экспорт карты:** Карта сохраняется во временный PNG-файл с высоким DPI (300).
6.  **Сборка отчета:** ReportLab создает PDF-страницу нужного размера и ориентации, размещает рамку, текстовые блоки (заголовок, автор, организация, дата) и отрисованную карту.
7.  **Загрузка результата:** PDF-отчет загружается в бакет `geo-reports` S3.
8.  **Публикация статуса:** Результат работы (успех со ссылкой на файл, либо ошибка) отправляется в Kafka-топик `geo.print.results`. Временная папка удаляется.

## ⚙️ Установка и запуск

### Локальный запуск (в виртуальном окружении):

1.  Установите системные зависимости для Cartopy (для Ubuntu/Debian):
    ```bash
    sudo apt-get install libproj-dev proj-data proj-bin libgeos-dev
    ```
2.  Создайте виртуальное окружение и установите зависимости:
    ```bash
    python3 -m venv venv
    source venv/bin/activate
    pip install -r requirements.txt
    ```
3.  Настройте переменные окружения в файле `.env` или передайте их при запуске:
    *   `KAFKA_BOOTSTRAP_SERVERS` (default: `localhost:9092`)
    *   `MINIO_ENDPOINT` (default: `localhost:9000`)
    *   `MINIO_ACCESS_KEY`
    *   `MINIO_SECRET_KEY`
    *   `TITILER_URL` (default: `http://localhost:8000`)
4.  Запустите воркер:
    ```bash
    python main.py
    ```

### Запуск через Docker:

Сборка образа:
```bash
docker build -t geoprint-worker .
```
Запуск контейнера:
```bash
docker run --env-file .env geoprint-worker
```
