# GeoInfoSystem Connector (QGIS Plugin)

Профессиональный плагин для QGIS, обеспечивающий полную интеграцию с микросервисной платформой **GeoInfoSystem**. Плагин позволяет работать с геоданными платформы напрямую из настольного приложения, поддерживая многоуровневую иерархическую структуру проектов (Слои ➔ Папки ➔ Векторы и Растры), 3D-визуализацию рельефа, прямое стриминговое отображение COG и асинхронный импорт Shapefile-архивов.

##  Основные возможности

*   **Централизованная конфигурация домена (`core/config.py`):**
    *   Все внешние точки подключения и адреса подсистем строятся на базе константы базового домена `DEFAULT_DOMAIN = "localhost"`.
*   **OAuth2 Авторизация:** Безопасный вход через системный браузер (SSO) с использованием протокола PKCE и автоматическим обновлением токенов (Refresh Token).
*   **Иерархия проектных данных (Layer ➔ GeoFolder ➔ Objects):**
    *   Полная идентичность древовидной структуре веб-интерфейса системы.
    *   **Слой (`Layer`):** Логический контейнер (тип `VECTOR`, `RASTER`, `TERRAIN`).
    *   **Папка (`GeoFolder`):** Поддержка бесконечной вложенности папок внутри слоев.
    *   **Векторы и Растры:** Нативное распределение объектов по папкам и корням слоев проекта.
*   **Двусторонняя синхронизация векторов:** 
    *   Загрузка точек, линий и полигонов напрямую из `geodata-service`.
    *   **Автоматическая трансформация координат (CRS):** Прозрачное преобразование геометрии из системы координат исходного слоя в WGS 84 (EPSG:4326) при синхронизации на сервер.
    *   **Нормализация типов:** Автоматическое приведение геометрии к Multi-типам (`MultiPolygon`, `MultiLineString`) для обеспечения целостности данных в PostGIS.
    *   Отправка изменений одной кнопкой (**Sync**) с поддержкой пакетного редактирования.
*   **Импорт векторных архивов Shapefile (.zip):**
    *   Новая кнопка **"Upload Shapefile (.zip)"** для загрузки локальных архивов напрямую в S3 MinIO.
    *   **2-Step Verify & Import:** Плагин отправляет архив на верификацию (`dataType: "SHAPEFILE"`), а затем запускает фоновый процесс конвертации GDAL в PostGIS (`taskType: "SHAPEFILE_TO_GEOJSON"`).
*   **Профессиональная работа с растрами (COG):** 
    *   Прямое подключение **Cloud Optimized GeoTIFF** через драйвер GDAL `/vsicurl/`.
    *   Использование **Presigned URLs** с автоматической Nginx-маршрутизацией (`/imagery/cog/` и `/terrain/cog/`), что позволяет QGIS читать только необходимые тайлы (отрисовка файлов объемом 1–10 ГБ происходит мгновенно).
*   **Поддержка 3D-рельефа (Terrain DEM):** Подключение слоев рельефа (**Terrain COG**) через `/vsicurl/`. Автоматическая настройка Hillshade-рендерера и активация свойств Elevation для нативного использования в 3D View QGIS.
*   **Прямая 2-этапная загрузка растров (Direct Raster Upload):**
    *   Загрузка локальных GeoTIFF слоев в систему через кнопку **"Upload Raster"**.
    *   **Техническая реализация:** Использование Presigned URLs для `PUT`-запросов напрямую в MinIO со стриминговой передачей (`io.BufferedReader`).
    *   После загрузки плагин запускает двухэтапную цепочку верификации и задачи оптимизации (`RAW_GEOTIFF_OPTIMIZE`) для публичного TiTiler COG вещания.
*   **Zero Trust & Security:** Работа исключительно через API Gateway (порт 80) и SSO с поддержкой PKCE.

## 📁 Архитектура плагина

```text
pyqgis-plugin/
├── core/
│   ├── config.py                 # Единый конфигурационный файл констант домена (localhost)
│   ├── auth.py                   # Логика OAuth2 PKCE, временный HTTP-сервер для Callback
│   ├── api_client.py             # REST-клиент для взаимодействия с микросервисами
│   └── layer_factory.py        # Фабрика слоев: CRS трансформация, COG/vsi-подключения
├── ui/
│   ├── dock_widget.py            # Панель проводника (Layers ➔ Folders ➔ Objects)
│   ├── raster_upload_dialog.py   # Диалог 2-этапной загрузки GeoTIFF растров
│   └── shapefile_upload_dialog.py# Диалог 2-этапной загрузки Shapefile архивов (.zip)
├── main.py                       # Точка входа, инициализация плагина
└── metadata.txt                  # Метаданные расширения QGIS
```

## 🚀 Установка

1. Скопируйте папку `pyqgis-plugin` в директорию профиля QGIS:
   * **Windows:** `%APPDATA%\QGIS\QGIS3\profiles\default\python\plugins`
   * **Linux:** `~/.local/share/QGIS/QGIS3/profiles/default/python/plugins`
   *(Или используйте подготовленный архив `pyqgis-plugin.zip` для установки через "Управление модулями ➔ Установить из ZIP")*.
2. Перезапустите QGIS.
3. Активируйте в меню: `Модули -> Управление модулями -> GeoInfoSystem Connector`.

## 🛠️ Использование

1. Нажмите иконку **GeoInfoSystem Connector**. Пройдите авторизацию в браузере.
2. В появившейся панели разверните нужный проект:
   * **Глобальные группы:** `Imagery Layers` (глобальные COG) и `Terrain Layers` (рельеф).
   * **Проектное дерево:** Логические слои `Layer` (VECTOR / RASTER) ➔ Папки `GeoFolder` ➔ Объекты.
3. **Загрузка данных:**
   * Используйте **"Upload Shapefile (.zip)"** для загрузки векторных архивов.
   * Используйте **"Upload Raster"** для выгрузки локальных GeoTIFF растров.
4. **Синхронизация:**
   * Внесите изменения во вложенных векторных слоях QGIS и нажмите **"Sync"** для сохранения геометрий и атрибутов в облаке.

## 🔍 Диагностика и отладка

Плагин ведет подробный лог сетевых вызовов, верификаций и Presigned URLs:
1. Нажмите на иконку диалога/сообщений в нижнем правом углу QGIS.
2. Перейдите на вкладку **GeoInfoSystem**.
3. Там отображаются сырые URL от API, логи трансформации маршрутов Nginx и статус инициализации провайдеров GDAL.

## 📋 Требования

* **QGIS:** 3.18 или выше (рекомендуется 3.34+ LTR).
* **Сеть:** Доступ к домену `localhost` (порт 80 для Gateway и порт 9000 для S3 MinIO).
