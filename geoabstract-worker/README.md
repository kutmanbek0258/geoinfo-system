# GeoAbstract Worker

Специализированный воркер на Python для обработки и оптимизации геопространственных данных с использованием библиотек GDAL/OGR.

## Основные возможности

Воркер асинхронно считывает события из очереди Kafka и выполняет следующие задачи:

* **Верификация данных (`VERIFY_FILE`):**
  * Проверка форматов входных файлов (GeoTIFF, Sentinel-2 ZIP/TAR, Landsat-8 ZIP/TAR, DEM, NetCDF).
  * Извлечение метаданных: проекция (CRS), гео-координатный охват (BBox), размеры сетки.
  * Анализ спектральных каналов и многомерных переменных (субдатасетов) для конфигурации импорта.
* **Sentinel-2 Processing (`SENTINEL_COG`):**
  * Извлечение спектральных каналов из `.SAFE` (ZIP) архивов.
  * Сборка многоканального растра, выполнение цветового синтеза (пресетов RGB) или расчет индексов вегетации (NDVI, NDWI, EVI и др.).
  * Сохранение результата в оптимизированный облачный формат **Cloud Optimized GeoTIFF (COG)**.
* **Landsat-8 Processing (`LANDSAT_COG`):**
  * Обработка, синтез каналов и расчет индексов по аналогичному с Sentinel-2 пайплайну для данных Landsat-8.
* **Оптимизация GeoTIFF (`RAW_GEOTIFF_OPTIMIZE` / `TERRAIN_COG`):**
  * Конвертация обычных растров в тайловые COG-файлы с поддержкой сжатия Deflate и автоматическим вычислением статистик.
* **Импорт NetCDF (`NETCDF_COG`):**
  * Чтение субдатасетов и извлечение конкретной многомерной переменной (например, температуры, влажности).
  * Автоматическое назначение географической привязки (WGS-84 / EPSG:4326) при ее отсутствии в метаданных и сборка в COG.

## Технологический стек

- **Python 3.10+**
- **GDAL 3.6+ / OGR** — базовый движок обработки растров.
- **Kafka-python** — для асинхронного взаимодействия.
- **MinIO Python SDK** — для скачивания исходников и загрузки готовых COG-файлов.

## Настройка (Environment Variables)

| Переменная | Описание | Значение по умолчанию |
| :--- | :--- | :--- |
| `KAFKA_BOOTSTRAP_SERVERS` | Адрес брокера Kafka | `kafka:9092` |
| `KAFKA_TOPIC` | Топик для событий растровых задач | `geoabstraction.raster.events` |
| `KAFKA_GROUP_ID` | Группа потребителей Kafka | `geoabstract-worker-group` |
| `MINIO_ENDPOINT` | Эндпоинт S3-хранилища | `minio:9000` |
| `MINIO_ACCESS_KEY` | Access Key MinIO | `minio_access_key` |
| `MINIO_SECRET_KEY` | Secret Key MinIO | `minio_secret_key` |
| `DEFAULT_WEB_RGB_MAX` | Ограничение максимальной яркости при Web RGB | `8` |

## Взаимодействие с TiTiler

Готовые растровые COG-слои загружаются воркером обратно в бакет `geo-abstraction-input` в папку `imagery-cog/`. Они обслуживаются динамическим тайлером **TiTiler**, который читает пиксели напрямую из MinIO "на лету" без распаковки файлов на диске.
