# Оптимизация загрузки тяжелых растровых данных (Direct S3 Upload)

**Статус: РЕАЛИЗОВАНО**

## Описание решения
Внедрена архитектура **Direct S3 Upload** на основе **Presigned URLs**. Это позволило исключить микросервисы Java из процесса пересылки тяжелых бинарных данных, передавая файлы напрямую из браузера пользователя в хранилище MinIO.

### Преимущества:
1. **Стабильность**: Отсутствие таймаутов на уровне API Gateway при загрузке файлов объемом 2ГБ+ (протестировано до 10ГБ).
2. **Экономия ресурсов**: Снижение потребления RAM и нагрузки на CPU микросервиса `geoabstraction-service`.
3. **Обратная связь**: Реализован плавный индикатор прогресса (Progress Bar) на фронтенде.

## Схема взаимодействия

1. **Frontend**: Вызывает `GET /upload/presigned-url`, передавая имя файла.
2. **geoabstraction-service**: Генерирует временную PUT-ссылку через MinIO SDK.
3. **Frontend**: Выполняет `axios.put(url, file)` напрямую в MinIO.
   - Запросы проксируются через `nginx-proxy` (маршрут `/minio/`).
   - Nginx автоматически подменяет заголовок `Host` на `minio:9000` для валидности S3-подписи.
4. **Frontend**: После HTTP 200 от MinIO вызывает `POST /jobs/confirm`.
5. **geoabstraction-service**: Проверяет физическое наличие файла в бакете и отправляет задачу в Kafka для Python-воркеров.

## Настройка инфраструктуры
Для корректной работы механизма в `nginx-proxy` настроены:
- Проксирование API MinIO.
- Полная поддержка CORS для метода `PUT` и специфичных заголовков S3 (`x-amz-*`).
