# Стратегия передачи данных рельефа (Terrain) через COG GeoTIFF

## 1. Введение
Текущая реализация системы передает данные рельефа для QGIS через Cesium Quantized Mesh, упакованный в gzip через Nginx-proxy. Эта схема оптимизирована для веб-фронтенда (CesiumJS), но создает избыточную нагрузку и задержки в настольных ГИС. Новая стратегия предполагает использование **Cloud Optimized GeoTIFF (COG)** для прямой передачи в QGIS через **Presigned URLs** из MinIO.

## 2. Архитектурные изменения

### 2.1 terrain-worker (Python)
Воркер обработки рельефа будет модернизирован для параллельной генерации двух форматов:
1.  **Quantized Mesh**: Сохраняется для совместимости с CesiumJS.
2.  **COG GeoTIFF**: Создается из нормализованного DEM.

**Техническая реализация:**
- После генерации `normalized.tif` (WGS84), выполняется команда:
  ```bash
  gdal_translate normalized.tif terrain_cog.tif -of COG -co COMPRESS=DEFLATE -co PREDICTOR=2 -co BIGTIFF=YES
  ```
- Загрузка `terrain_cog.tif` в MinIO по ключу: `terrain-cog/{jobId}.tif`.
- В Kafka-сообщении о завершении (`READY`) передается `cogObjectKey`.

### 2.2 geoabstraction-service (Java/Spring Boot)
Сервис управления данными расширяется для поддержки метаданных COG.

**Изменения в БД:**
- Таблица `geoabstraction.terrain_layers`: добавить колонку `cog_object_key` (VARCHAR).

**Изменения в коде:**
- `TerrainLayer.java`: добавить поле `cogObjectKey`.
- `GeoAbstractionServiceImpl.java`:
    - При обновлении статуса задачи сохранять `cogObjectKey` в сущность слоя.
    - В методах получения слоев генерировать временный **Presigned URL** для `cogObjectKey`.

### 2.3 QGIS Plugin (Python)
Плагин QGIS обновляется для приоритетного использования COG.

**Логика работы:**
- При получении списка слоев рельефа, плагин проверяет наличие `cogUrl`.
- Если `cogUrl` доступен, создается `QgsRasterLayer` с использованием сетевого пути.
- Слой автоматически помечается как `Elevation` в свойствах QGIS для возможности использования в 3D-представлении.

## 3. Преимущества стратегии
1.  **Производительность**: QGIS читает только необходимые фрагменты данных через HTTP Range-запросы.
2.  **Точность**: COG сохраняет исходные значения высот в Float32 без потерь, в отличие от квантованной сетки.
3.  **Масштабируемость**: Снятие нагрузки с Nginx-proxy за счет прямой передачи данных из объектного хранилища.
4.  **Совместимость**: Веб-интерфейс продолжает работать с привычным форматом тайлов.

## 4. План реализации
1.  Миграция базы данных (добавление колонки).
2.  Обновление `terrain-worker` (генерация и загрузка COG).
3.  Обновление `geoabstraction-service` (API и Presigned URLs).
4.  Доработка плагина QGIS.
