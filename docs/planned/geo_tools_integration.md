# **План интеграции оперативных ГИС-инструментов в веб\-клиент ГеоИнфоСистемы**

## **1\. Концепция и архитектурное обоснование**

Настоящий план описывает стратегию развития веб\-клиента **ГеоИнфоСистемы** в качестве легкого «операционного дашборда». Вся сложная ГИС-аналитика, верификация топологии и геопроцессинг остаются в зоне ответственности настольного клиента (QGIS). Веб-интерфейс, в свою очередь, фокусируется на скорости, навигации, мониторинге и легком полевом сборе данных.

## **2\. Дорожная карта (Roadmap) внедрения**

Реализация разбита на четыре логических этапа по принципу нарастания сложности и приоритета для бизнес-пользователей.

### **Этап 1: Инструменты оперативных измерений**

* **Умная линейка и площадь:** Интеграция инструментов измерения в OpenLayers с обязательным включением функции «снаппинга» (прилипания) к узлам векторных тайлов (MVT), транслируемых через pg\_tileserv.  
* **Динамический профиль высот:** Инструмент для построения 2D-графика разреза рельефа по нарисованной линии, использующий данные от terrain-worker и 3D-движка Cesium.  
* **Построение буферных зон:** Простой UI для создания радиусов вокруг выбранных точек интереса (POI) для оценки зон покрытия или поражения.

### **Этап 2: Инструменты визуального мониторинга ДЗЗ**

* **Инструмент сравнения «Шторка» (Swipe Tool):** Реализация функционала для визуального сравнения двух растровых слоев (например, снимков Sentinel-2 до и после события).  
* **Временной слайдер (Time Slider):** Элемент управления для переключения между историческими снимками или состояниями слоев для оценки динамики изменений.  
* **Точечный опрос растра:** Получение точного значения пикселя (например, индекса NDVI) по клику без скачивания исходного Cloud Optimized GeoTIFF (COG).

### **Этап 3: Аннотирование и коммуникация (Redlining)**

* **Слой графических пометок:** Создание отдельного временного векторного слоя (на базе GeoJSON) для рисования полигонов, линий, маркеров и текстовых комментариев поверх официальной карты.  
* **Шеринг состояния (Permalink):** Генерация коротких ссылок, кодирующих текущие координаты, уровень зума и список включенных слоев с их прозрачностью, для быстрой передачи текущего вида карты коллегам.  
* **Быстрая печать:** Экспорт текущего экстента карты в PDF/PNG в один клик с добавлением масштабной линейки, стрелки севера и легенды активных слоев.

### **Этап 4: Легкий полевой сбор данных**

* **Адаптивные карточки (Identify Tool):** Сводное информационное окно при клике, агрегирующее данные геометрии из geodata-service, атрибуты и привязанные файлы из MinIO/OnlyOffice.  
* **Формы быстрого обновления:** UI для изменения статусов объектов из выпадающего списка и загрузки фотографий с камеры напрямую в S3-совместимое хранилище через Presigned URLs (Direct Upload).

## **3\. Технические требования и стек**

Разработка должна вестись с использованием текущего технологического стека системы, чтобы обеспечить бесшовную интеграцию.

| Компонент | Технология | Роль в интеграции   |
| :---- | :---- | :---- |
| **Фронтенд** | Vue.js 3, OpenLayers, CesiumJS | Отрисовка UI-компонентов, обработка событий мыши (измерения, шторка, рисование аннотаций). |
| **Векторные данные** | pg\_tileserv, PostGIS | Обеспечение привязки (снаппинга) к MVT-тайлам при измерениях, фильтрация данных на клиенте. |
| **Растровые данные** | GeoServer, COG, MinIO | Поставка тайлов для временного слайдера и шторки, отдача данных для точечного опроса пикселей. |

## **4\. Пример архитектуры компонента (Фронтенд)**

Для реализации инструментов измерения со снаппингом к MVT-слоям в OpenLayers рекомендуется следующий паттерн взаимодействия:

// Пример инициализации взаимодействия для инструмента измерений  
import { Interaction, Draw, Snap } from 'ol/interaction';  
import VectorLayer from 'ol/layer/Vector';

// 1\. Инициализация слоя для временной геометрии (измерений)  
const measureLayer \= new VectorLayer({ source: measureSource });

// 2\. Добавление инструмента рисования  
const drawInteraction \= new Draw({  
    source: measureSource,  
    type: 'LineString'  
});

// 3\. Добавление снаппинга к рабочему MVT слою (критически важно для точности)  
const snapInteraction \= new Snap({  
    source: businessDataMvtSource // источник из pg\_tileserv  
});

map.addInteraction(drawInteraction);  
map.addInteraction(snapInteraction);
