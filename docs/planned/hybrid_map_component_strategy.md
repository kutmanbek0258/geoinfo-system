# Стратегия перехода на гибридный картографический компонент (MVT + GeoJSON)

## 1. Введение и Обоснование

На текущем этапе в системе существуют два параллельных компонента: `MapComponent.vue` (для работы с небольшими объемами данных через GeoJSON) и `MapComponentMVT.vue` (для визуализации больших массивов данных через векторные тайлы). 

**Проблема:** Разделение на два компонента приводит к дублированию 80% кода UI и логики, усложняет поддержку и создает разрывный пользовательский опыт.

**Цель:** Объединить оба подхода в едином компоненте, который автоматически выбирает оптимальный формат данных в зависимости от масштаба (Zoom) и задачи (просмотр или редактирование).

---

## 2. Сравнение архитектурных подходов

| Характеристика | Текущая (Раздельная) | Целевая (Гибридная) |
| :--- | :--- | :--- |
| **Компоненты** | `MapComponent` и `MapComponentMVT` | Единый `UnifiedMapComponent` |
| **Производительность** | Ограничена либо объемом данных, либо точностью | Максимальная: MVT для объема, GeoJSON для точности |
| **Синхронизация** | Ручная между слоями | Автоматическая через общую логику Vuex |
| **Поддержка** | Сложная (правки вносятся в двух местах) | Упрощенная (единый код UI и инструментов) |

---

## 3. Целевая архитектура "Smart Layer Manager"

В основе нового компонента будет лежать логика интеллектуального переключения источников данных.

### 3.1. Режимы работы
1.  **Global View (Zoom 2 - 16):**
    *   Используются **MVT-тайлы**.
    *   Минимальная нагрузка на клиент.
    *   Мгновенная отрисовка сотен тысяч объектов.
2.  **Precision View (Zoom 17+):**
    *   Автоматическая подгрузка **GeoJSON** для видимой области.
    *   MVT-слой скрывается (opacity: 0) для предотвращения дублирования.
    *   Обеспечивается "живое" взаимодействие с объектами.

### 3.2. Механизм редактирования
*   При переходе в режим редактирования система всегда использует GeoJSON-источник.
*   Измененный объект отображается в `tempSource` поверх всех слоев до момента подтверждения сохранения и обновления кэша тайлов.

---

## 4. План реализации (Фазы)

### Фаза 1: Унификация визуализации и стилей
1.  **Shared Stylers:** Вынос логики `parseStyle` в универсальный утилитарный класс, работающий с любыми типами фич OpenLayers.
2.  **Unified Tooltips:** Объединение логики всплывающих подсказок (Hover/Click) для работы как с RenderFeature (тайлы), так и с Feature (GeoJSON).

### Фаза 2: Интеграция GeoJSON в MVT-компонент
1.  Добавление в `MapComponentMVT` полноценного `VectorSource` для хранения объектов проекта.
2.  Реализация `LayerSwitchController`, отслеживающего событие `moveend` карты и принимающего решение о подгрузке данных.

### Фаза 3: Миграция инструментов
1.  Перенос специфичных функций из `MapComponent.vue` (например, сложные инструменты снаппинга или специфичные для GeoJSON фильтры).
2.  Оптимизация `ObjectDetails` для работы в едином контексте.

### Фаза 4: Удаление технического долга
1.  Удаление файла `MapComponent.vue`.
2.  Переименование (при необходимости) и очистка `MapComponentMVT.vue` от временных "костылей".

---

## 5. Ожидаемые преимущества
1.  **Для пользователя:** Карта всегда работает быстро и точно, независимо от того, смотрит он на всю страну или редактирует один забор.
2.  **Для разработчика:** Единый стек технологий и инструментов (Measurements, Swipe, Draw) в одном месте.
3.  **Для системы:** Снижение нагрузки на API при использовании MVT на обзорных масштабах и снижение нагрузки на PostGIS за счет точечных GeoJSON запросов на крупных масштабах.
