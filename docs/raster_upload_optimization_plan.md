# План оптимизации загрузки тяжелых растровых данных

## Текущая проблема
В настоящее время загрузка файлов (Sentinel-2, Landsat 8, GeoTIFF) происходит по цепочке:
`Client -> Nginx Proxy -> API Gateway -> geoabstraction-service -> MinIO`.

**Недостатки:**
1. **Избыточное копирование:** Файл пересылается через несколько промежуточных узлов, что увеличивает время загрузки.
2. **Нагрузка на JVM:** Обработка многогигабайтных Multipart-запросов в Java-сервисах приводит к высокому потреблению памяти и частым циклам Garbage Collection.
3. **Таймауты:** API Gateway и прокси-серверы имеют ограничения по времени ожидания, что приводит к обрывам загрузки на медленных соединениях.
4. **Отсутствие контроля:** Пользователь не видит реального прогресса загрузки, так как браузер сообщает о завершении, когда файл дошел до Gateway, а не до хранилища.

## Целевая архитектура (Direct S3 Upload)
Оптимизация предполагает использование механизма **Presigned URLs**, позволяющего фронтенду загружать файлы напрямую в MinIO, минуя промежуточные микросервисы.

### Схема взаимодействия:
1. **Frontend**: Запрашивает у `geoabstraction-service` подписанную ссылку (Presigned URL) для конкретного файла.
2. **geoabstraction-service**: Генерирует уникальную временную ссылку через MinIO SDK.
3. **Frontend**: Выполняет `PUT` запрос напрямую в MinIO (через Nginx).
4. **Frontend**: После успешной загрузки уведомляет `geoabstraction-service` о готовности файла.
5. **geoabstraction-service**: Проверяет наличие файла и запускает стандартный процесс обработки (Kafka -> Worker).

## Этапы реализации

### 1. Backend (`geoabstraction-service`)
- Добавить эндпоинт `GET /api/geo-abstraction/upload/presigned-url`:
  - Вход: `filename`, `content_type`.
  - Выход: `url` (подписанная ссылка), `objectKey` (будущий путь к файлу).
- Добавить эндпоинт `POST /api/geo-abstraction/jobs/confirm`:
  - Вход: данные о загруженном объекте и параметры задачи (каналы, индексы).
  - Логика: Валидация наличия объекта в MinIO и создание записи `GeoAbstractJob` со статусом `QUEUED`.

### 2. Frontend (`Vue.js`)
- Обновить `geo-abstraction.service.ts`:
  - Добавить методы для получения Presigned URL и подтверждения загрузки.
  - Реализовать метод прямой загрузки через `axios` с отслеживанием `onUploadProgress`.
- Обновить `SatelliteImageryUploadDialog.vue`:
  - Заменить прямую отправку FormData на двухэтапный процесс (URL -> Upload -> Confirm).
  - Добавить визуальный индикатор прогресса (ProgressBar).

### 3. Инфраструктура (`Nginx` / `Docker`)
- Настроить CORS в MinIO для разрешения запросов с домена приложения.
- Добавить в `nginx-proxy` маршрут до API MinIO (порт 9000), чтобы избежать проблем с Cross-Origin и SSL.

## Ожидаемые показатели
- **Скорость**: Сокращение времени от начала загрузки до начала обработки на 40-60%.
- **Стабильность**: Возможность загрузки файлов объемом до 5-10 ГБ без изменения настроек таймаутов Gateway.
- **Масштабируемость**: Высвобождение ресурсов Java-микросервисов для обработки бизнес-логики, а не пересылки байтов.
